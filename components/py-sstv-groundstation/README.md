# Наземная станция SSTV на Python/C++

Приложение наземной станции для приема сигналов с реальных спутников (например, МКС) и их декодирования в изображения. Этот проект близко имитирует реальные операции наземной станции и работает с фактическими космическими данными.

## Цель

Принимать сигналы с реальных спутников (например, МКС) и декодировать их в изображения, используя Python в качестве основной системы с дополнительными компонентами на C++ для операций, чувствительных к производительности.

## ✨ Особенности

- **Прием сигнала**: Использование SDR-приемников (RTL-SDR V4 и др.) для захвата радиосигналов
- **Декодирование SSTV**: Декодирование SSTV-сигналов с использованием библиотеки pysstv
- **Обработка в реальном времени**: Обработка потоков живого сигнала и декодирование в реальном времени
- **Сохранение изображений**: Сохранение декодированных изображений для дальнейшего анализа
- **Сканирование диапазона**: Автоматический поиск активных частот
- **Оптимизация производительности**: Дополнительные модули на C++ для вычислительно сложных задач

## 🌐 Архитектура

- **Основная система на Python**: Обрабатывает основную логику приложения, пользовательский интерфейс и координацию
- **Компоненты на C++** (необязательно): Низкоуровневая обработка аудиопотока (цифровые фильтры для очистки сигнала)
- **Используемые библиотеки**:
  - `pysstv` - для декодирования SSTV
  - `rtlsdr` / `pyrtlsdr` - для приема сигналов от SDR-приемников (RTL-SDR V4)
  - `numpy`, `scipy` - для обработки сигналов
  - `matplotlib` - для визуализации

## 🚀 Начало работы

### Необходимые условия
- Python 3.8 или выше
- SDR-приемник (RTL-SDR V4 рекомендуется)
- Антенна для диапазона 145-146 МГц (VHF)
- Соответствующая аппаратная настройка для радиоприема

### Установка

#### Шаг 1: Установка зависимостей Python

```bash
pip install -r requirements.txt
```

Или вручную:
```bash
pip install pysstv rtlsdr pyrtlsdr numpy scipy matplotlib
```

#### Шаг 2: Установка драйверов RTL-SDR

**Для Windows:**

1. Скачайте драйверы с официального сайта: https://www.rtl-sdr.com/
2. Распакуйте архив в удобную папку (например, `C:\rtl-sdr`)
3. Добавьте путь к DLL в системную переменную PATH
4. **Важно:** Используйте утилиту **Zadig** для установки драйера WinUSB:
   - Скачайте Zadig: https://zadig.akeo.ie/
   - Запустите от имени администратора
   - В меню выберите: Options → List All Devices
   - Выберите ваше RTL-SDR устройство
   - Выберите драйвер: **WinUSB**
   - Нажмите "Replace Driver" или "Install Driver"

**Для Linux:**

```bash
sudo apt-get install rtl-sdr librtlsdr-dev
sudo modprobe rtlsdr
```

**Для macOS:**

```bash
brew install rtl-sdr
```

### Проверка подключения

```bash
# Показать доступные RTL-SDR устройства
python src/main.py --list-devices

# Пример вывода:
# Найдено RTL-SDR устройств: 1
#   Устройство 0: Realtek RTL2838UHIDIR
```

### Запуск

#### Быстрый старт - прием с МКС:

```bash
python src/main.py --sdr --frequency 145.800 --duration 60
```

#### Прием с автоматическим декодированием:

```bash
python src/main.py --sdr --frequency 145.800 --duration 60 --auto-decode
```

#### Сканирование диапазона:

```bash
python src/main.py --scan --scan-start 145.0 --scan-end 146.0
```

#### Декодирование аудиофайла:

```bash
python src/main.py --audio recording.wav
python src/main.py --audio recording.wav --detect
```

## 🧩 Модули

### Декодер SSTV (`sstv_decoder.py`)
Обрабатывает декодирование SSTV-сигналов в изображения с использованием библиотеки pysstv.

### Интерфейс SDR (`sdr_interface.py`)
Взаимодействует с RTL-SDR для приема радиосигналов:
- Инициализация устройства
- Настройка частоты, усиления, частоты дискретизации
- FM демодуляция
- Запись в WAV формат
- Сканирование диапазона

## 📖 Команды и параметры

### Основные режимы

| Команда | Описание |
|---------|----------|
| `--audio <файл>` | Декодировать аудиофайл |
| `--sdr` | Режим приема с RTL-SDR |
| `--scan` | Сканирование диапазона |
| `--list-devices` | Показать доступные устройства |

### Параметры SDR

| Параметр | Короткий | По умолчанию | Описание |
|----------|----------|--------------|----------|
| `--device-index` | `-d` | `0` | Индекс RTL-SDR устройства |
| `--frequency` | `-f` | `145.800` | Частота в МГц |
| `--gain` | `-g` | `30` | Усиление (0-50 дБ) |
| `--duration` | `-t` | `30` | Длительность записи (сек) |

### Параметры сканирования

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `--scan-start` | `145.0` | Начало диапазона (МГц) |
| `--scan-end` | `146.0` | Конец диапазона (МГц) |
| `--scan-step` | `0.001` | Шаг сканирования (МГц) |

### Общие параметры

| Параметр | Описание |
|----------|----------|
| `--output`, `-o` | Путь сохранения (изображение/аудио) |
| `--detect` | Обнаружить SSTV сигнал в аудио |
| `--auto-decode` | Автодекодирование после записи |

## 🛰️ Работа с реальными спутниками

### Частоты для приема

| Спутник | Частота (МГц) | Режим | Описание |
|---------|---------------|-------|----------|
| **МКС (ISS)** | 145.800 | SSTV/APRS | Любительская радиостанция |
| NOAA-18 | 137.9125 | APT | Метеоспутник |
| NOAA-19 | 137.1000 | APT | Метеоспутник |
| Meteor-M2 | 137.100 | LRPT | Метеоспутник |

### Расписание SSTV МКС

Проверьте веб-сайты для расписания передач SSTV с Международной космической станции:
- https://www.ariss.org/
- https://www.heavens-above.com/
- https://n2yo.com/

### Рекомендации по антенне

Для диапазона 145 МГц (МКС):
- **Дипольная антенна** - простая и эффективная
- **Yagi 3-5 элементов** - направленная, высокое усиление
- **Квадрокоптерная** - круговая диаграмма
