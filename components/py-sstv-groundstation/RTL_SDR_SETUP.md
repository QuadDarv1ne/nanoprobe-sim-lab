## 🔍 Что такое RTL-SDR V4

**RTL-SDR V4** - это программно-определяемая радиосистема (SDR) на базе чипа RTL2832U.

### Технические характеристики V4:

| Параметр | Значение |
|----------|----------|
| Частотный диапазон | 500 кГц - 1.766 ГГц |
| Максимальная частота дискретизации | 3.2 MSPS |
| Разрешение АЦП | 8 бит |
| Интерфейс | USB 2.0 |
| Улучшения V4 | Улучшенная развязка, TCXO, переключатель bias-tee |

### Отличия V4 от V3:
- ✅ Улучшенная печатная плата
- ✅ Лучшая развязка по питанию
- ✅ Встроенный TCXO (термостабильный генератор)
- ✅ Переключатель bias-tee для активных антенн
- ✅ Улучшенная защита от помех

## 🪟 Установка на Windows

### Шаг 1: Установка драйверов

#### 1.1 Скачайте драйверы

Перейдите на https://www.rtl-sdr.com/ и скачайте последнюю версию драйверов.

Или используйте Zadig для автоматической установки:

#### 1.2 Установка через Zadig (РЕКОМЕНДУЕТСЯ)

1. **Скачайте Zadig:** https://zadig.akeo.ie/
2. **Запустите от имени администратора**
3. **Настройте Zadig:**
   - Меню: `Options` → `List All Devices` ✓
   - Отключите `Ignore Hubs or Composite Parent Devices`
4. **Выберите устройство:**
   - В выпадающем списке выберите "Bulk-In, Interface"
   - Или "RTL2838UHIDIR"
5. **Выберите драйвер:**
   - В поле драйвера выберите **WinUSB (v6.x.x.x)**
6. **Установите:**
   - Нажмите `Replace Driver` или `Install Driver`
   - Подтвердите установку

![Zadig настройка](https://rtl-sdr.com/wp-content/uploads/2015/08/zadig.jpg)

### Шаг 2: Установка библиотек Python

   - Добавить путь к папке с rtlsdr.dll

## 🐧 Установка на Linux

### Шаг 1: Установка пакетов

**Ubuntu/Debian:**
```bash
sudo apt-get update
sudo apt-get install rtl-sdr librtlsdr-dev libusb-1.0-0-dev
```

**Fedora:**
```bash
sudo dnf install rtl-sdr rtl-sdr-devel
```

**Arch Linux:**
```bash
sudo pacman -S rtl-sdr
```

### Шаг 2: Настройка правил udev

Создайте файл правил:
```bash
sudo nano /etc/udev/rules.d/88-rtlsdr.rules
```

Добавьте содержимое:
```
# RTL-SDR V4
ATTR{manufacturer}=="Realtek", ATTR{product}=="RTL2838UHIDIR", MODE:="0666", GROUP:="plugdev"
```

Примените правила:
```bash
sudo udevadm control --reload-rules
sudo udevadm trigger
```

### Шаг 3: Добавление пользователя в группу

```bash
sudo usermod -aG plugdev $USER
# Или для некоторых дистрибутивов:
sudo usermod -aG dialout $USER
```

**Перезайдите в систему** для применения изменений.

### Шаг 4: Проверка

```bash
# Проверка обнаружения устройства
rtl_test -t

# Проверка в системе
lsusb | grep RTL
```

### Шаг 5: Установка Python библиотек

```bash
pip install rtlsdr pyrtlsdr
```

## 🛰️ Запуск SSTV станции

### Быстрый старт - прием МКС

```bash
# Базовая команда
python src/main.py --sdr --frequency 145.800 --duration 60

# С автоматическим декодированием
python src/main.py --sdr --frequency 145.800 --duration 60 --auto-decode

# С настройкой усиления (максимальное)
python src/main.py --sdr --frequency 145.800 --duration 60 --gain 50

# Сохранение в конкретный файл
python src/main.py --sdr --frequency 145.800 --duration 60 --output mks_recording.wav
```

### Параметры для разных спутников

| Спутник | Частота (МГц) | Команда |
|---------|---------------|---------|
| МКС (ISS) | 145.800 | `--frequency 145.800` |
| NOAA-18 | 137.9125 | `--frequency 137.9125` |
| NOAA-19 | 137.1000 | `--frequency 137.1000` |

## 📡 Прием МКС

### Подготовка

1. **Узнайте расписание пролетов МКС:**
   - https://www.heavens-above.com/
   - https://n2yo.com/
   - https://www.ariss.org/

2. **Подготовьте антенну:**
   - Направьте на предполагаемую траекторию
   - Используйте направленную антенну для лучшего приема

3. **Проверьте оборудование:**
   - Подключите RTL-SDR за 5 минут до пролета
   - Проверьте уровень сигнала

### Команда для приема

```bash
# Запись во время пролета
python src/main.py --sdr \
    --frequency 145.800 \
    --duration 120 \
    --gain 45 \
    --output iss_pass_$(date +%Y%m%d_%H%M%S).wav \
    --auto-decode
```

### Советы по приему

- **Время:** Начинайте запись за 2-3 минуты до ожидаемого сигнала
- **Антенна:** Чем выше, тем лучше
- **Помехи:** Уберите электронику подальше
- **USB:** Используйте качественный кабель

## 🔧 Устранение проблем

### Проблема: Устройство не обнаруживается

**Решение:**
1. Проверьте подключение USB
2. Попробуйте другой USB порт (желательно USB 2.0)
3. Перезапустите Zadig и переустановите драйвер
4. Проверьте в Диспетчере устройств

### Проблема: Ошибка "No supported devices found"

**Решение:**
```bash
# Windows: Переустановите драйвер через Zadig
# Linux: Проверьте правила udev
lsusb | grep RTL
# Должно показать устройство

# Проверьте права доступа
ls -l /dev/bus/usb/*/
```

### Проблема: Слабый сигнал

**Решение:**
1. Увеличьте усиление: `--gain 50`
2. Используйте внешнюю антенну
3. Поднимите антенну выше
4. Используйте USB удлинитель

### Проблема: Много помех

**Решение:**
1. Добавьте ферритовые кольца на кабель
2. Отодвиньте от электроники
3. Используйте экранированный кабель
4. Попробуйте bias-tee (если антенна активная)

### Проблема: Ошибка Python библиотек

**Решение:**
```bash
# Переустановите библиотеки
pip uninstall rtlsdr pyrtlsdr
pip install rtlsdr pyrtlsdr

# Проверьте установку
python -c "from rtlsdr import RtlSdr; print('OK')"
```

### Проблема: Дрейф частоты

**Решение:**
- RTL-SDR V4 имеет TCXO, но при необходимости:
```python
# В sdr_interface.py добавьте коррекцию
sdr.set_freq_correction(ppm)
```
