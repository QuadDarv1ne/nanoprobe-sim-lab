# Отчёт: Автоматическое определение портов

**Дата:** 2026-04-08  
**Статус:** ✅ Выполнено  
**Версия:** 1.0.0

---

## Резюме

Реализована система автоматического определения свободных портов для всех сервисов проекта. Теперь можно запускать несколько экземпляров без конфликтов.

---

## Что реализовано

### 1. Утилита `utils/port_finder.py` ✅

**Функционал:**
- ✅ Сканирование используемых портов (Windows netstat, Linux/Mac ss)
- ✅ Поиск свободного порта для сервиса
- ✅ Поиск портов для нескольких сервисов одновременно
- ✅ Проверка конкретного порта
- ✅ Предложение нескольких портов
- ✅ Резервирование найденных портов (предотвращение race conditions)

**Диапазоны поиска:**

| Сервис | Приоритетный | Fallback |
|--------|--------------|----------|
| Backend (FastAPI) | 8000 | 8000-8049, 8100-8149 |
| Flask Frontend | 5000 | 5000-5049, 5100-5149 |
| Next.js Frontend | 3000 | 3000-3049, 3100-3149 |

**Тест:**
```bash
python utils/port_finder.py
```

**Результат теста:**
```
✅ Найденные порты:
  backend        : 8000
  flask          : 5001  ← автоматически выбран свободный
  nextjs         : 3000
```

## Использование

### Сценарий 1: Запуск нескольких экземпляров

```bash
# Терминал 1: Backend на 8000
python run_api.py

# Терминал 2: Backend на 8001 (автоопределение)
python run_api.py

# Терминал 3: Flask на 5001 (автоопределение)
python src/web/web_dashboard_unified.py
```

### Сценарий 2: Фиксированные порты

```bash
# В .env файле:
BACKEND_PORT=8000
FLASK_PORT=5000
NEXTJS_PORT=3000

# Запуск
python main.py flask
```

### Сценарий 3: Ручной выбор порта

```bash
python run_api.py --port 8080
```
